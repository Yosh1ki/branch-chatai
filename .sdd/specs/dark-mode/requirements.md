# 要件定義書

## 機能概要
Branchアプリにダークモードを実装し、ユーザーが明るい環境・暗い環境のどちらでも視認性を保って利用できるようにする。既存のマインドマップ型チャットUI（本筋・分岐・ミニマップ・設定画面）で一貫した配色体験を提供する。

## ユーザーストーリー
- ユーザーとして、画面テーマをダークモードに切り替えたい。なぜなら、夜間や長時間利用時の目の負担を減らしたいから。

## 機能要件
### 要件1：テーマ切り替え
- ユーザーがライト/ダークを切り替えられるUIを提供し、選択したテーマを即時に画面へ反映する。
- 受入基準：
  - [ ] 設定操作でライト/ダークのテーマを切り替えられること
  - [ ] 切り替え後、主要画面（チャット一覧、チャット詳細、設定）に即時反映されること

### 要件2：テーマ設定の保持
- ユーザーが選択したテーマを再訪時にも維持し、都度再設定が不要な状態にする。
- 受入基準：
  - [ ] ページ再読み込み後も選択テーマが保持されること
  - [ ] 再ログイン後もテーマ設定が意図どおり適用されること

### 要件3：主要UI要素の可読性担保
- ダークモード時にテキスト、背景、境界線、操作要素のコントラストを確保し、分岐構造の認知性を損なわないこと。
- 受入基準：
  - [ ] メッセージ本文、インラインタイトル、入力欄、ボタンが判読可能であること
  - [ ] 分岐線、ノード、ミニマップの強調表示が視認できること

## 非機能要件
- パフォーマンス：テーマ切り替え操作は体感的に遅延なく反映されること
- セキュリティ：テーマ設定の保存処理で認証・認可境界を越える他ユーザー設定の参照や更新が発生しないこと
