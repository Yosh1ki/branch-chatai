# 要件定義書

## 機能概要
`chats/[chatId]` 画面をキャンバスのように拡大・縮小・上下左右移動できるUIにする。現在のチャット表示は維持し、既存機能を壊さない。

## ユーザーストーリー
- ユーザーとして、`chats/[chatId]` を拡大・縮小しながら自由に移動したい。なぜなら、会話全体を俯瞰しながら細部も確認したいから。

## 機能要件
### 要件1：キャンバス操作の導入
- `chats/[chatId]` の画面全体をパン・ズーム可能なキャンバスとして扱う。
- 既存のチャット表示（ヘッダー、メッセージ表示など）はそのまま維持する。
- 受入基準：
  - [ ] 画面上でドラッグ操作により上下左右に移動できること
  - [ ] ホイール操作またはピンチ操作で拡大・縮小できること
  - [ ] 既存のチャット表示内容が消えたり崩れたりしないこと

### 要件2：操作性と制約
- 拡大・縮小の最小・最大倍率を設ける。
- パン操作時にコンテンツが極端に見失われない範囲で移動制限を設ける。
- 受入基準：
  - [ ] 拡大・縮小が指定範囲内に制限されること
  - [ ] パン操作時に一定範囲内で位置が制限されること

### 要件3：操作UIの補助
- 現在の拡大率を視覚的に把握できるUI（例：倍率表示）を用意する。
- 初期状態に戻す操作（リセット）を用意する。
- 受入基準：
  - [ ] 拡大率が画面上で確認できること
  - [ ] リセット操作で初期位置と倍率に戻ること

## 非機能要件
- パフォーマンス：パン・ズーム操作が滑らかで、UI操作が引っかからないこと
- セキュリティ：既存の認証・権限フローを変更しないこと
